# specode

Specification-driven workflow plugin for **Claude Code** and **CodeBuddy**.

This plugin runs the workflow on top of **advisory-only hooks** — every
hook injects guidance into the model's context (`exit 0` +
`additionalContext`) and never blocks tool calls. State binds to the
Claude `session_id` so multiple windows stay disambiguated, and the
six spec documents (`requirements.md` / `bugfix.md` / `design.md` /
`tasks.md` / `acceptance-checklist.md` / `implementation-log.md`)
remain the single source of truth.

## What you get

- **Session lifecycle bound to `claude_session_id`.** Every Claude
  Code session has its own state file at
  `~/.specode/sessions/<session_id>.json` with mode (active /
  readonly / ended), active spec slug, phase, lock state, and the
  current `pending_selector`. All writes are atomic.
- **Persistent session is the only mode.** `/specode:spec <requirement>`
  always creates a persistent session; `/specode:end` writes
  `mode=ended` so hooks immediately stop injecting on the next turn.
- **Seven advisory hooks (`exit 0`, never block):** SessionStart,
  two UserPromptSubmit handlers (selector / doc reminder + silent
  lock heartbeat), PreToolUse, Stop, PostToolUse Task, SessionEnd.
  Together they manage the session lifecycle, inject the right
  selector prompt at each phase-gate node, remind the model to
  keep docs and code in sync, drive the task-swarm orchestrator,
  and require a status footer on every active-spec turn.
- **Selector text is generated by the model.** The hook injects only
  *"render the X selector (type ?)"* metadata; the model formats the
  actual selector text per three skeletons (A single-select / B
  wizard / C multi-select). 11 fixed scenario constants live in
  `spec_session.py SELECTOR_PROMPTS`.
- **Status footer on every active turn:**
  ```
  ─── spec-mode ─── spec: <slug> | session: <8-prefix> | phase: <p> | /specode:end 退出
  ```
- **Document-first discipline.** UserPromptSubmit reminds the model
  to check whether the current input requires a doc edit before
  code; Stop reminds it to backfill docs if it touched code. Both
  are advisory text only.
- **`/specode:spec -h` fast-path** — hook intercepts and injects the
  full help text into context for the model to verbatim-print.
- **`spec-writer` agent** — physically isolated document author
  (tools: Read, Write, Edit, Grep, Glob; no Bash).

## Architecture

```
.claude-plugin/marketplace.json   ← single-plugin marketplace manifest
plugins/specode/
  .claude-plugin/plugin.json      ← plugin manifest
  hooks/hooks.json                ← 7 advisory hook handlers
  commands/                       ← /specode:spec, /specode:continue,
                                    /specode:end, /specode:status,
                                    /specode:task-swarm
  agents/                         ← task-swarm-{planner,coder,reviewer,
                                    validator}, spec-writer
  scripts/
    spec_vault.py                 ← 3-tier doc-root resolution +
                                    Obsidian vault detection
    spec_init.py                  ← spec scaffolding + atomic
                                    sessions / .config.json writes
    spec_session.py               ← business + hook subcommands;
                                    SELECTOR_PROMPTS constant library;
                                    list-specs for /specode:continue
    spec_lint.py                  ← 4 advisory lint rules
    spec_status.py
    task_swarm.py                 ← task-swarm CLI (init / plan /
                                    advance / writeback / heartbeat /
                                    resolve)
    task_swarm_{state,parse_md,outbox,writeback,prompt}.py
    run.sh / run.cmd              ← Python launchers
  skills/specode/
    SKILL.md                      ← spec-mode discipline contract
    references/                   ← workflow / lock-protocol /
                                    obsidian / prompts (selector
                                    library) / templates / iteration /
                                    task-swarm / task-swarm-example
  assets/templates/               ← seed templates
  tests/                          ← 152 pytest cases
```

## Install

### From GitHub

```sh
# Claude Code
claude plugin marketplace add https://github.com/qxbyte/specode
claude plugin install specode@specode

# CodeBuddy (verified on 2.97.1)
codebuddy plugin marketplace add https://github.com/qxbyte/specode
codebuddy plugin install specode@specode
```

### One-shot session (Claude Code only)

```sh
claude --plugin-url https://github.com/qxbyte/specode/archive/refs/heads/main.zip
```

### Local development

```sh
git clone https://github.com/qxbyte/specode.git
claude    --plugin-dir ./specode/plugins/specode
codebuddy --plugin-dir ./specode/plugins/specode
```

### Uninstall

```sh
claude plugin uninstall specode@specode
claude plugin marketplace remove specode
# optional: remove user-level state
rm -rf ~/.specode ~/.config/specode
```

## Usage

```
/specode:spec <requirement>             # create a persistent spec session
/specode:continue [slug]                # resume / switch
/specode:status                         # show current session state
/specode:end                            # end the session (preserves docs)

/specode:spec --set-vault <path>        # bind Obsidian vault
/specode:spec --set-root <path>         # bind a non-vault doc root
/specode:spec --detect-vault            # list installed vaults
/specode:spec --vault-status            # show current doc root & source
/specode:spec -h                        # full help (hook fast-path)
```

Once an active session is open:

- Every user prompt gets a status block + selector reminder (when at
  a phase-gate) + document-first reminder injected into model
  context.
- Every model turn ends with a code-doc sync reminder.
- The model must render selectors per the three skeletons and emit
  the status footer.

## Global bypass

```sh
SPECODE_GUARD=off   # short-circuits all hooks to exit 0
```

For debugging only.
