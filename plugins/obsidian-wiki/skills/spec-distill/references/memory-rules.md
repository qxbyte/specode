# MEMORY 维护规则

> `spec-distill sync` 更新各系统 `MEMORY.md` 时遵循本规则。核心原则：**只追加，不重排，不改旧行，不改 H1**。

---

## 1. 头部模板（首次 sync 到某系统时写入）

首次 sync 到某系统（包括新建系统和旧系统头部迁移）时，将 MEMORY 头部统一改为：

```text
> 区域索引：[[10-Work/README|10-Work]]

# Knowledge Base Index — <系统名>

> Auto-generated by spec-distill. 手动编辑请保留表格结构。
> 知识库路径: 10-Work/知识库/<系统名>
```

**必须去掉**以下写死机器路径的行（跨平台兼容）：
```text
> Vault: C:/Users/Public.DESKTOP-KVO3HU1/Documents/Notes
```

**不得改动**：H1 标题行（`# Knowledge Base Index — <系统名>`）、表格内容、知识关联图内容。

---

## 2. 首次 sync 到已有旧系统（旧头迁移）

若系统 MEMORY 仍是旧 `kn-indexer` 生成的头部，执行头部迁移：

**识别旧头特征**（满足其一即触发迁移）：
- 含 `> Auto-generated by kn-indexer` 行
- 含 `> Vault: C:/` 行

**迁移操作**（Read → 精确替换，不动其余内容）：

将以下旧头块：
```text
> Auto-generated by kn-indexer. Do not edit manually.
> Vault: C:/Users/Public.DESKTOP-KVO3HU1/Documents/Notes
> 知识库路径: 10-Work/知识库/<系统名>
```

替换为新头块：
```text
> Auto-generated by spec-distill. 手动编辑请保留表格结构。
> 知识库路径: 10-Work/知识库/<系统名>
```

同时在文件最顶部确保有 `> 区域索引：[[10-Work/README|10-Work]]`（若已有则不重复写）。

**现有语料参照**：`10-Work/知识库/新收付（fin）/MEMORY.md` 目前仍是旧头（kn-indexer），首次 sync 到该系统时执行上述迁移。

## 3. 更新规则（sync 追加索引时）

### 3.1 知识文档索引表——追加行

每新建一篇知识文档，在 `## 知识文档索引` 表末尾追加一行：

```markdown
| [[<文档标题>]] | <一句摘要（≤30字）> | <需求号, 多个用逗号分隔> |
```

**示例**（参照现有 fin MEMORY）：
```markdown
| [[结算授权-收付登记号查询授权]] | 非见费出单按收付登记号维度查询与授权，三表同步更新paymentComCode | 121659 |
| [[SfCreditMain字段说明]] | PAYSTATUS枚举、CNY_PAY_AMOUNT归属、表关联关系 | 123000 |
```

**约束**：
- 不重排已有行的顺序
- 不修改已有行的内容（含摘要和关联需求）
- 文件名与 Obsidian wikilink 一致（不带 `.md` 后缀）

### 3.2 需求反向索引表——追加或更新行

对本次 sync 的需求号，在 `## 需求反向索引` 表中：

- 若该需求号**不存在**→ 末尾追加一行：
  ```markdown
  | <需求号> | <需求名称> | [[<文档1>]], [[<文档2>]] |
  ```
- 若该需求号**已存在**→ 只修改其"关联知识"列，追加新文档的 `[[..]]`，不改其他列

**示例**（参照现有 fin MEMORY）：
```markdown
| 114371 | 收付个人数据脱敏需求 | [[收付个人数据脱敏]], [[收付系统接口规范V3.12]], [[银行账号加密重构-双参数并行处理]], [[sflosstransinfo字段说明]] |
| 121659 | 非见费出单收付登记号授权 | [[结算授权-收付登记号查询授权]], [[往来关系上报机制]], [[Q01承保送收付处理逻辑]], [[收付登记号弹窗-收付款人账号调用链路]], [[收付系统接口规范V3.12]] |
```

**约束**：
- 不重排已有行顺序
- 不改已有需求行的"需求名称"列

### 3.3 知识关联图——用户确认后追加

`## 知识关联图` 代码块中的 ascii 图**不自动修改**。若本次 sync 新增了有关联关系的文档，用 `AskUserQuestion` 展示拟追加的图节点，用户确认后再 append 到代码块末尾。

**不得**：重写整张关联图、删除现有节点。

## 5. 操作顺序与安全

1. **改前必读**：`Read MEMORY.md` 取得最新内容，再做精确字符串替换（不盲写）。
2. **批量前备份**：同系统批量写入多篇文档前，tar 备份该系统目录到 `~/Library/Caches/spec-distill-backup-<ts>/`（Windows 退化为同盘临时目录）。
3. **写后记 wiki-log**：每次 sync 后 append `00-Index/_system/wiki-log.md`：
   ```
   - YYYY-MM-DD HH:MM spec-distill <项目目录> -> 10-Work/知识库/<系统>/ (需求号: X, 新建: N 篇)
   ```
4. **唯一可写路径**：`10-Work/知识库/<系统>/`，SpecIn 原文永不修改。
